title: Variables

ReVoman supports several types of variables across all template formats:

- **Variables**, e.g., `{{variable-key}}` or `{{ variable-key }}` (spaces are trimmed)
- **Nested variables**, e.g., `{{variable-key{{nested-variable-key}}}}`
- **[Dynamic variables](https://github.com/salesforce-misc/ReVoman/blob/master/src/main/kotlin/com/salesforce/revoman/internal/postman/DynamicVariableGenerator.kt)**, e.g., `{{$randomUUID}}`, `{{$randomEmail}}`
- **[Custom Dynamic variables](/ReVoman/guides/custom-dynamic-variables/)**

:::note
The `{{variable}}` syntax is shared by both Postman and JetBrains `.http` files. ReVoman's variable resolution engine works identically for both formats.
:::

## Variable resolution precedence

In the case of collision between variable keys, the precedence order to derive a value to replace any key is:

1. [Custom Dynamic variables](/ReVoman/guides/custom-dynamic-variables/)
2. [Dynamic variables](https://github.com/salesforce-misc/ReVoman/blob/master/src/main/kotlin/com/salesforce/revoman/internal/postman/DynamicVariableGenerator.kt)
3. Environment built during execution (via scripts, hooks, or handlers)
4. Dynamic Environment supplied through config
5. Environment supplied as a file path through config (Postman or `http-client.env.json`)

:::tip
**Variable naming convention:** For Dynamic variables which get populated from Java code, use a `$` sign within a variable to indicate it needs to be populated during manual testing, e.g.: `{{$unitPrice}}`.
:::
