ReVoman executes API workflow templates through a JVM program, translating manual testing into automation without resistance.
It supports **multiple template formats** — bring your existing Postman collections or JetBrains `.http` files.

![Postman Run](../../assets/images/postman-run.png)

![Manual to Automation](../../assets/images/manual-to-automation.png)

It strikes a balance between _flexibility_ provided by low-level tools like
[REST Assured](https://rest-assured.io/) or [Cucumber](https://cucumber.io/) and _ease of use_
provided by UI tools like [Postman](https://www.postman.com/) or [JetBrains HTTP Client](https://www.jetbrains.com/help/idea/http-client-in-product-code-editor.html).

The fundamental principle is **Template-Driven Testing** — your existing API workflow templates become the test specification.

:::note
NO licensed Postman SDKs are used inside this project. This is written ground up natively in JVM.
:::

## Supported Template Formats

<CardGrid>
  <Card title="Postman Collections" icon="document">
    Use exported Postman collection JSON files as templates. Includes support for pre-request scripts, test scripts, and `npm` modules via GraalVM.
  </Card>
  <Card title="JetBrains HTTP Files" icon="document">
    Use `.http` files from JetBrains IDEs. Includes support for response handlers and pre-request handlers, with the HTTP Client JavaScript API.
  </Card>
</CardGrid>

## Features

<CardGrid>
  <Card title="Multi-Format" icon="list-format">
    Supports Postman collections and JetBrains `.http` files. A clean `TemplateParser` abstraction makes it straightforward to add more formats.
  </Card>
  <Card title="Type-Safe" icon="approve-check">
    Bridges the gap between strongly-typed JVM code and flexible JSON with built-in marshalling and unmarshalling.
  </Card>
  <Card title="Low Code" icon="pencil">
    Approximately 89% less code compared to traditional integration/E2E tests. Low cognitive complexity and transparent.
  </Card>
  <Card title="CI/CD Ready" icon="rocket">
    Works like any JVM library — plug into JUnit tests or integration tests. No extra setup needed for CI/CD.
  </Card>
</CardGrid>

## Installation

<Tabs>
  <TabItem label="Maven">
    ```xml
    <dependency>
        <groupId>com.salesforce.revoman</groupId>
        <artifactId>revoman</artifactId>
        <version>0.82.0</version>
    </dependency>
    ```
  </TabItem>
  <TabItem label="Gradle Kts">
    ```kotlin
    implementation("com.salesforce.revoman:revoman:0.82.0")
    ```
  </TabItem>
  <TabItem label="Bazel">
    ```python
    "com.salesforce.revoman:revoman"
    ```
  </TabItem>
</Tabs>

:::caution
Minimum Java version required: **21**
:::
