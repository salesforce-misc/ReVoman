## Multi-Format Template Support

ReVoman supports multiple API workflow template formats through a clean `TemplateParser` abstraction:

- **Postman collections** (`.json`) — familiar to most developers, with full script and folder support
- **JetBrains `.http` files** — native to IntelliJ-based IDEs, with response handler and pre-request handler support

Both formats can even be used together in a single test run. Adding new formats requires implementing a single interface.

## Low Learning Curve

The learning curve is way-way-low compared to any other automation frameworks out there (like Cucumber, REST Assured, etc.), as there is no proprietary syntax or template formats required for this tool. Familiarity with Postman or JetBrains HTTP Client gets you a long way in understanding this tool. Playing with the existing integration tests and writing a couple of hands-on tests should get you started quickly.

---

## Low Setup and Maintenance 'Tax'

Unlike tools like Cucumber, which demand a high setup and maintenance 'Tax' with a lot of glue code, this tool has no strict requirements for wiring your templates with code. They are loosely coupled — the relative path to the template file in the config is the only link between them, so you are free to organize your templates and code independently.

---

## CI/CD Integrability

- ReVoman is like any JVM library that you can plug into any JVM program/test (e.g., JUnit tests or Integration tests).
- Apart from adding a dependency in the build tool, there is **no extra setup needed** to execute these tests with ReVoman in CI/CD.

---

## Up-to-date Templates that Live Along with Code in VCS

- A nice side effect is, this lets the templates always stay up to date and the entire collection guards each **check-in** in the form of a test suite augmenting manual testing.
- Any day, you can find up-to-date Postman collections or `.http` files for every feature you need to test, right in your VCS (Git) along with your code. Developers can import these templates directly into Postman or open them in IntelliJ for manual testing. This comes in very handy during a team blitz/bug-bash.
- The low-code test itself acts like a live Runbook, stitching scattered API templates, that can explain a scenario to anyone doing an exploratory testing.

---

## Unified Framework for Automating Persona-based Manual Testing

- ReVoman brings a **Unified & Simplified Test strategy** across the mul-**T**-verse (Integration Tests, E2E Tests, and Manual testing) for any API.
- The automation stays as close as possible to Persona-based Manual testing, leading to Transparency and better Traceability of issues.
- This forces engineers to think like **API-first customers** while writing tests.
- **Test Data setup:** You can use ReVoman for test data setup too. This eliminates the need for different teams to write their own internal utilities for data setup.
- **E2E Test** can even reside outside the Service repo, as long as it can hit the service API.
