<section class="rv-band rv-how">
  <header class="rv-band__head">
    <p class="rv-section-marker">// how the engine works</p>
    <h2 class="rv-band__title">A graph in. A Rundown out.</h2>
    <p class="rv-band__sub">
      Think of the <a href="https://learning.postman.com/docs/collections/running-collections/intro-to-collection-runs/">Postman Collection Runner</a> walking a collection step by step — but inside a Java/Kotlin program, resolving the dependencies between calls. Today that metadata is serialized in the Postman&nbsp;V2/V3 schema; reading Postman-schema metadata no more makes this "a Postman tool" than emitting JSON makes a service "a JSON tool."
    </p>
  </header>
  <ol class="rv-steps3">
    <li class="rv-move">
      <span class="rv-move__idx">01</span>
      <h3 class="rv-move__title">Feed it the metadata</h3>
      <p class="rv-move__body">An <a href="executable-api-metadata.html">API-Graph</a>: the workflow of interdependent calls, with the order and the data dependencies between them.</p>
    </li>
    <li class="rv-move">
      <span class="rv-move__idx">02</span>
      <h3 class="rv-move__title">It executes the graph</h3>
      <p class="rv-move__body">Deterministically, on the JVM — resolving data dependencies, handling auth, polling for async outcomes, marshalling <a href="type-safety.html">type-safe</a> payloads.</p>
    </li>
    <li class="rv-move">
      <span class="rv-move__idx">03</span>
      <h3 class="rv-move__title">Get back a Rundown</h3>
      <p class="rv-move__body">A structured record of everything that happened. <a href="rundown.html">Rundown</a><code>.toJson(VERBOSITY)</code> is the Context Information an agentic Node consumes.</p>
    </li>
  </ol>
