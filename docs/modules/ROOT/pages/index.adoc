= ReṼoman

ReṼoman is an *API Orchestration Engine* for the JVM. You describe a workflow of interdependent API calls as *executable API metadata*; ReṼoman executes that graph — resolving data dependencies between calls, handling auth, polling, and type-safe payloads — and returns a structured *Rundown* of everything that happened. This makes it a building block for *Agentic API Orchestration*, with API test automation as one powerful application.

ReṼoman comes from the API-first SaaS company, *Salesforce*.

Current version: {revoman-version}

[NOTE]
====
*What ships today vs. what's on the roadmap.* ReṼoman is honest about the line between the two.

*Today* — an API Orchestration Engine that deterministically executes executable API metadata (serialized in the Postman V2/V3 schema) on the JVM, resolving data dependencies, auth, and polling; returns a `Rundown` whose `toJson(VERBOSITY)` is structured *Context Information*; offers type safety, dynamic hooks, timing metrics, and the Postman `pm.*` script APIs.

*Roadmap* (direction, *not* shipped — never read these as present-tense capabilities): format-agnosticism beyond Postman V2/V3; MCP tool-definition autogen; dynamic coupling of agentic workflows to API-Graphs. These are clearly labeled wherever they appear.
====

NOTE: NO licensed Postman SDKs are used inside this project. This is written ground up natively in JVM.

== From manual testing to JVM-native execution

ReṼoman reads *executable API metadata* and executes it on the JVM. Today that metadata is serialized in the Postman V2/V3 collection schema — a schema choice in the same spirit as picking JSON over XML. A service that serializes its data as JSON is not "a JSON tool"; likewise, ReṼoman reading Postman-schema metadata is not "a Postman tool." It is decoupled from Postman the app/SDK and written ground-up natively for the JVM.

To get an intuition for it, think of how the https://learning.postman.com/docs/collections/running-collections/intro-to-collection-runs/[Postman Collection Runner] walks a collection step by step. ReṼoman performs that same deterministic, step-by-step execution of an API-Graph — but inside a Java/Kotlin program or test, so your manual testing translates into automation without any loss or resistance. And it goes further.

== A hybrid tool

The engine strikes a balance between the _flexibility_ of low-level tools like https://rest-assured.io/[REST Assured] or https://cucumber.io/[Cucumber] and the _ease of use_ of UI tools like https://www.postman.com/[Postman].

image::hybrid-tool.png[]

== Fundamental principle: execute without compromising manual testing

The fundamental principle of the engine is to execute your API metadata on the JVM *without compromising on manual testing*. Because the engine executes the very metadata you already author for manual trials, the same metadata stays ready to import back into Postman and try out by hand. JVM automation and manual testing share one source of truth instead of drifting apart.
